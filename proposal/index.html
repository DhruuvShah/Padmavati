Option B — Implementation Plan

SKU Auto-Generation + Barcode/QR · Phase by phase

Prepared for Padmavati Corporation · Prepared by Dhruv Shah · 25 June 2026

This is the execution plan for the approved Option B scope — SKU auto-generation and automatic Barcode/QR codes, ₹50,000, 46 hours. No new scope has been added here; every line below maps directly to the cost breakdown already quoted. This document just sequences it into phases with clear goals and exit criteria, and adds one no-cost phase (0) to keep the delivered build honest about what's actually paid for.

Quick status check, since we just audited the live app: 0% of Option B exists today. No SKU field, no barcode, no QR — confirmed by searching the entire codebase. Everything below starts from zero. Meanwhile, two features got built that were never part of either quotation — the catalogue access-request/OTP system and Parties management. Phase 0 addresses that directly.

Phase 0 · No Cost

Scope Hygiene — Disable, Don't Delete

Make the delivered admin panel match exactly what's been paid for, without throwing away finished work.
Goal
Hide out-of-scope features from the admin UI
Effort
~2 hours, unbilled
Exit Criteria
Sidebar shows only paid features; extras are visibly "coming soon"

What Gets Disabled

  • Access Requests — the OTP-gated catalogue sharing/approval workflow
  • Parties — the customer code-name management page

How It Behaves

Both nav items stay visible but greyed out and unclickable. Hovering shows a short tooltip instead of an error — something like:

Access Requests Coming in a future update Parties Available in a later phase

Nothing is deleted — the code, routes, and database tables stay exactly as built. The moment either feature is actually in scope (Option A, or its own add-on), it's a one-line flip back on. No rebuild, no re-testing from scratch.

Sequencing, At a Glance

Phase 0 → Phase 1 → Phase 2 → Phase 3, in that order. Phase 0 runs first and costs nothing — it's a UI-level switch, not new engineering, so it doesn't push out the timeline. Phase 2 cannot start before Phase 1 finishes, since it generates codes from the SKU Phase 1 creates. Phase 3 runs last by definition — it's what proves Phases 1–2 actually hold up.
